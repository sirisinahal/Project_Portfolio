Sprint 2




Analysis of the O*NET website


The O*NET database contains a large number of variables that represent job and worker characteristics, such as skill needs. The professions are defined by the O*NET-SOC taxonomy, and the Content Model specifies which information is gathered. With data from the working public, the Data Collection programme brings these frameworks to life.





Sprint 2 provided an excellent gaming experience. In this case, we had to research the O*NET website and design an uniform schema for the Skills Taxonomy. Sprint 3




AWS Services - API Gateway, Lambda, S3


These services were chosen because they are highly scalable, durable, safe and secure, and easy to manage clusters. Amazon API Gateway is an AWS service that allows you to create, publish, maintain, monitor, and secure REST, HTTP, and WebSocket APIs at any scale. API developers may construct APIs that interact with AWS or other online services, as well as data stored in the AWS Cloud.

AWS Lambda is a serverless computing service that runs code in response to events and maintains the underlying computing resources autonomously. Amazon Simple Storage Service (Amazon S3) is online storage. Amazon S3 may be used to store and retrieve any quantity of data from anywhere on the internet at any time.


In Sprint 3, I delved into AWS services, investigating API Gateway, Lambda, and S3. Furthermore, I uploaded the files to an S3 bucket, wrote code to create API calls for both skills and job titles using the POST method, and granted permission to internal teams to use these API calls to extract the appropriate data using API Gateway. Analyzed and updated the code to meet the needs of all error codes issued while accessing API requests using the POSTMAN tool.

Sprint 4




BERT and FuzzyWuzzy


The BERT algorithm was chosen since it was created by Google and is the most profitable NLP model available for context matching.

The FuzzyWuzzy technique is mostly used for string matching, we utilised it to match job titles.



I studied several String matching algorithms and NLP models for matching Job titles and competencies for Sprint 4. Later in my research, I discovered that the BERT and FuzzyWuzzy algorithms were the greatest fit for my needs.I ran both the BERT and FuzzyWuzzy algorithms on a sample data set of around 2000 job titles and skills, manually compared the results, and calculated accuracy using a confusion matrix, and the results led us to the conclusion that the data needed to be cleaned and normalised to improve the algorithm's accuracy.

Sprint 5





I focused on boosting the algorithm's accuracy for Sprint 5. These findings lead us to conclude that the True positives and False positives data for the manually cleaned and uncleaned data had changed.
